# ST Deck Agent 用户手册

> **适用版本：** 与仓库 `main` 同步的 Web 产品（Deck 生成）。  
> **服务对象：** 需要使用 ST 品牌合规 PPT 的内部同事。  
> **部署与运维：** 请参阅仓库根目录 [README.md](../README.md)。

---

## 1. 产品简介

**ST Deck Agent** 是面向 STMicroelectronics 内部的品牌合规演示文稿生成工具。您在浏览器中描述需求（并可上传参考文档），AI Agent 会按 ST PPT 品牌规范自动构建 `.pptx`，并在页面中提供逐页 PNG 预览与下载。

**典型用途：**

- 根据产品资料或会议纪要快速产出汇报页  
- 将长文档浓缩为多页「一页一核心信息」的 deck  
- 在已有初稿基础上通过「在线编辑」微调版式与文案（仍由 Agent 改 `build.py` 并重新渲染）

**当前主能力：** **Deck 生成**（一键成稿 + 可选对话模式 + 在线编辑）。  
**规划中（尚未在默认环境开放）：** **Deck 焕新**——上传自有 `.pptx` 仅做视觉标准化，不改文案；详见 [PRD-DECK-REFRESH.md](PRD-DECK-REFRESH.md)。

## 2. 开始之前

### 2.1 访问地址与访问口令

1. 向工具管理员获取 **Cloud Run 服务 URL**（生产服务名一般为 `st-deck-agent`，区域 `asia-east1`）。  
2. 若环境启用了 `ACCESS_TOKEN`，页面右上角会出现 **「访问口令」** 输入框：  
   - 输入管理员发放的口令后，浏览器会保存在本地（`localStorage`），后续请求自动携带 `X-Access-Token`。  
   - 预览图与下载链接也可通过 URL 参数 `access_token=` 访问（用于 `<img>` / 下载，无法自定义请求头时）。  
3. 未配置口令时，仅依赖 Cloud Run 是否对外匿名开放；**生产环境强烈建议启用口令**。

### 2.2 使用须知（首次打开弹窗）

每个浏览器 **会话** 首次进入时，需阅读并点击 **「我已阅读并同意」**：

| 要点 | 说明 |
|------|------|
| 行为准则 | 使用须遵守 STMicroelectronics《行为准则》（Code of Conduct）。 |
| 保密资料 | **不得**上传、粘贴或引用涉密、保密、出口管制或未公开的内部资料。 |
| AI 合规培训 | 使用前须完成《人工智能法律合规意识》（*Legal Awareness in Artificial Intelligence*）培训；违规使用后果由用户自行承担。 |

界面语言为 **中文 / EN / 日本語** 时，上述三条文案会对应切换。

## 3. 界面概览

| 区域 | 作用 |
|------|------|
| **顶栏** | 品牌标识、访问口令（若启用）、**界面语言**切换 |
| **左侧 · 撰写需求** | 需求正文、示例 chips、页数、幻灯片语言、参考文档、对话模式开关、生成按钮 |
| **右侧 · 工作台** | 待命说明 / 实时日志 / 完成后预览与下载 / 在线编辑或继续对话 |

**界面语言** 只影响按钮与提示文案；**幻灯片语言**（中文 / English / 日本語）决定 Agent 写入 PPT 的标题、信息条与正文语言（除非您在需求中明确要求双语）。

## 4. 生成 PPT（推荐流程）

### 4.1 撰写需求

在文本框中说明：

- **受众**（客户 / 领导 / 内部技术）  
- **主题与结构**（单页摘要 vs 多页故事线）  
- **必须出现的数据、产品名、结论**  
- 可选：**公开网页 URL**（Agent 可抓取页面截图用于幻灯片）

可点击 **「试试」** 下方示例 chip 快速填入模板，再按实际情况修改。

**快捷键：** `Ctrl + Enter`（macOS：`⌘ + Enter`）开始生成。

### 4.2 页数

- 可选 **1～6 页**（上限由管理员通过 `MAX_PAGES` 配置，页面加载时从 `/api/config` 读取）。  
- **页数提示（界面文案）：** 建议同一主题先生成 **多页**，再通过 **在线编辑** 或本机 PowerPoint 合并，内容往往比硬挤在一页更丰富。  
- **仅选 1 页时：** Agent 会按「高管摘要单页」处理——避免把三页故事塞进一页（例如滥用三栏卡片）；更适合一页一结论 + 少量要点或左图右文结构。

### 4.3 参考文档（可选）

支持拖拽或点击上传，常见类型：**PDF、Word、PPT、Excel、CSV、TXT、Markdown、PNG/JPG**。

| 限制项 | 默认值 |
|--------|--------|
| 单文件大小 | 10 MB |
| 单次总大小 | 30 MB |
| 文件个数 | 5 个 |

超出限制时接口会返回英文错误说明，请删减文件后重试。

### 4.4 生成过程与停止

点击 **「生成 PPT」** 后：

1. 工作台进入 **运行中**，展示思考过程、工具调用与状态。  
2. Agent 按四步工作：**理解与收集 → 套用 ST 品牌 → 构建并渲染 → 自检与修正**。  
3. 运行中可将主按钮变为 **「停止」**，中断当前请求（已生成的会话文件可能不完整）。  
4. 完成后状态为 **完成**，显示页数、**下载 .pptx** 链接及逐页预览图。

单次 Agent 运行有服务端超时（默认约 **9 分钟**）。页数多或需求复杂时，可拆成多次生成或减少页数。

## 5. 对话模式（Beta）

打开 **「对话模式」** 后，主按钮变为 **「开始讨论」**。

| 阶段 | 行为 |
|------|------|
| 首轮 | Agent **只规划**：给出大纲、追问缺失信息，**不会**立刻生成 `deck.pptx`。 |
| 后续 | 在 **「继续对话」** 中补充说明；当您明确表示确认（如「确认」「OK」）后，Agent 才开始 BUILD。 |
| 完成后 | 与一键模式相同：预览 + 下载；对话模式下 **不显示**「在线编辑」区块（请继续在对话中说明修改）。 |

**请注意：**

- 对话状态保存在 **服务实例内存** 中，Cloud Run 需 **`--max-instances 1`** 才能稳定多轮对话。  
- 实例重启或会话超过 **24 小时** TTL 后，对话上下文会丢失。  
- 功能标为 **Beta**，适合需要反复对齐大纲的场景；日常汇报仍推荐默认 **一键生成**。

## 6. 在线编辑（一键模式）

在一键生成 **成功产出 deck** 后，工作台下方出现 **「在线编辑」**：

1. 用自然语言描述修改（例如：「标题改为…」「信息条改为深蓝底白字」）。  
2. 点击 **「应用修改」**；Agent 在同一工作区内修改 `build.py`、重新运行并更新预览。  
3. 可多次编辑；下载链接会指向最新 `deck.pptx`。

编辑同样受访问口令、限流与运行超时约束。

## 7. 下载与预览

| 项目 | 说明 |
|------|------|
| **文件名** | 优先使用 Agent 写入的 `deck_meta.json`（主题 + 日期），否则从需求首行推导，形如 `主题-YYYY-MM-DD.pptx`。 |
| **预览图** | 服务端用 LibreOffice + Chromium 渲染，字体为 **Liberation Sans**（Arial 度量替代）。 |
| **本机打开** | 生成的 `.pptx` 仍声明 **Arial**；在您本机 PowerPoint 上与预览的细微差异属正常现象，**以本机打开效果为准**。 |
| **外发材料** | Agent 规则要求对外演示 deck 使用品牌 **closing / 商标页** 等合规要素（见内部 Skill 文档）；请在定稿前自行核对。 |

会话文件在服务器上默认保留约 **24 小时** 后自动清理；请及时下载。浏览器里保存的 session id 不能代替口令，也不能在过期后恢复文件。

## 8. 合规与内容规范（使用者责任）

- 仅用于 **内部授权场景**；输出内容须您自行核对事实与数字。  
- **禁止**输入或上传保密、个人敏感或未脱敏客户数据。  
- Agent 遵循 `workspace_template` 内 ST PPT 品牌 Skill（色板、Title-Only 版式、20pt 信息条、对比度等），但 **不能替代** 法务或品牌部门的最终审批。  
- 工具 **不会**使用 AI 生成图片充当产品实拍；图片一般来自上传或 URL 截图。

## 9. 使用限制（默认）

| 项目 | 默认值 |
|------|--------|
| 每 IP 每分钟请求数 | 10（生成 / 编辑 / 对话均计入） |
| 单次最大页数 | 6 |

| 单次运行超时 | 约 540 秒 |

触发限流时会提示稍后再试。若频繁超时，请减少页数、缩短需求或拆分任务。

## 10. 常见问题（FAQ）

**Q：提示 Unauthorized？**  
A：检查右上角访问口令是否正确；或联系管理员确认 `ACCESS_TOKEN` 是否已轮换。

**Q：生成失败或长时间无结果？**  
A：查看工作台红色错误信息；简化需求、减少页数后重试。持续失败请联系管理员查看 Cloud Run 日志。

**Q：刷新页面后还能下载吗？**  
A：若未超过 24 小时且同一会话仍在服务器上，且浏览器仍保存 session id，有可能继续访问；**可靠做法是在完成后立即下载**。

**Q：对话模式第二轮无响应？**  
A：可能命中多实例部署或会话过期；请改用一键模式或联系管理员确认 `max-instances=1`。

**Q：预览里文字换行和 PowerPoint 不一致？**  
A：预览为近似渲染，排版以本机 PowerPoint 为准。

**Q：Deck 焕新在哪里？**  
A：产品规划中，默认环境 **未开放** Tab；实现后将作为独立模块上线（不改文案、仅视觉标准化）。进展见 [PRD-DECK-REFRESH.md](PRD-DECK-REFRESH.md)。

**Q：管理员 / 部署问题？**  
A：参阅 [README.md](../README.md)；Cloud Shell 磁盘不足、Cloud Run 启动失败等见分支上的 [TROUBLESHOOTING.md](TROUBLESHOOTING.md)（合并后可在 `main` 直接访问）。

## 11. 英文速查（Quick reference）

| Item | Detail |
|------|--------|
